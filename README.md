## Дипломный проект. Задание 1: Юнит-тесты

### Автотесты для проверки программы, которая помогает заказать бургер в Stellar Burgers

### Реализованные сценарии

Созданы юнит-тесты, покрывающие классы `Bun`, `Burger`, `Ingredient`, `Database`

Процент покрытия 100% (отчет: `htmlcov/index.html`)

### Структура проекта

Diplom_1/

### Запуск автотестов

**Установка зависимостей**

> `$ pip install -r requirements.txt`

**Запуск автотестов и создание HTML-отчета о покрытии**

>  `$ pytest --cov=praktikum --cov-report=html`

## Описание тестов

### `tests/test_bun.py` — **TestBun**
| Тест-кейс                                                        | Что проверяет                                                                                   |
|------------------------------------------------------------------| ----------------------------------------------------------------------------------------------- |
| Параметризованное создание булочки с разными названиями и ценами | Корректное сохранение имени и цены, в том числе латиница/кириллица/спецсимволы и большие числа. |
| Геттеры возвращают значения нужных типов                         | `get_name()` → `str`, `get_price()` → `float`.                                                  |

### `tests/test_burger.py` — **TestBurger**
| Тест-кейс                                 | Что проверяет                                                                                                                                                               |
| ----------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Установка булочки                         | `set_buns()` присваивает булочку объекту бургера.                                                                                                                           |
| Добавление ингредиента                    | `add_ingredient()` добавляет ингредиент в конец списка.                                                                                                                     |
| Удаление ингредиента по индексу           | `remove_ingredient(index)` удаляет нужный элемент.                                                                                                                          |
| Перемещение ингредиента (параметризовано) | `move_ingredient(from, to)` корректно меняет порядок (из начала в конец, из конца в начало, внутренние перестановки).                                                       |
| Подсчёт цены (параметризовано, с моками)  | `get_price()` корректно суммирует: цена бургера = 2×цена булочки + сумма цен ингредиентов. Проверяются сценарии: без ингредиентов, с одним, с несколькими, с большой ценой. |
| Формирование чека                         | `get_receipt()` возвращает чек с правильным форматированием, списком ингредиентов и итоговой ценой.                                                                         |

### `tests/test_ingredient.py` — **TestIngredient**
| Тест-кейс                                | Что проверяет                                                                            |
| ---------------------------------------- | ---------------------------------------------------------------------------------------- |
| Параметризованное создание ингредиента   | Корректное сохранение типа, названия и цены, в т.ч. спецсимволы/кириллица/большие числа. |
| Геттеры возвращают значения нужных типов | `get_type()` → `str`, `get_name()` → `str`, `get_price()` → `float`.                     |

### `tests/test_database.py` — **TestDatabase**
| Тест-кейс                                   | Что проверяет                        |
| ------------------------------------------- | ------------------------------------ |
| `available_ingredients()` возвращает список | Тип результата — `list`.             |
| `available_buns()` возвращает список        | Тип результата — `list`.             |
| Список доступных ингредиентов не пустой     | В базе есть хотя бы один ингредиент. |
| Список доступных булочек не пустой          | В базе есть хотя бы одна булочка.    |
| У булочек валидные цены                     | У каждой булочки `price > 0`.        |
| У ингредиентов валидные цены                | У каждого ингредиента `price > 0`.   |
